Why Not MCP?

MCP (Model Context Protocol) is great for fine-grained tool integration, but the token economics tell a different story for bulk workloads.

MCP Approach

Each tool call is a round-trip through the agent. Describe 3,000 files? That is 3,000 agent interactions — each one burning context tokens for the request, the tool schema, and the response. The agent is doing procedural work it was never designed for.

Agentic Tool Approach

A single invocation kicks off the entire pipeline. Local models handle all 3,000 files autonomously. The agent pays only for the invoke command and reading the manifest. That is 3,000 local model calls at zero agent token cost.
